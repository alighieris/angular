Formulários e Validators

Formulários são uma solução para receber e registrar o input de usuários, em diferentes tipos de aplicações. O Angular oferece dois tipos de abordagem para lidar com esses inputs através de formulários. Ambos capturam os eventos de inputs a partir das views, validam, criam um "form model" e um "data model" e dispõem de maneiras de identificar e acompanhar mudanças nos inputs.
Formulários Reativos e Orientados a templates processam e gerenciam dados de formas distintas, cada um oferecendo diferentes vantagens na implementação:

  • Formulários Reativos: fornecem acesso direto e explícito à camada do modelo do objeto de formulário. Comparado ao form Orientado a templates, os Reativos são mais robustos: são mais escaláveis, reutilizáveis e testáveis. Indicado para uso em aplicações onde formulários são partes chave e/ou aplicações construídas com patterns reativos.
  • Formulários orientados a template: se valem de diretivas nos templates para criar e manipular à camada do modelo do objeto. Eles são mais diretos no processo de implementação, mas não são tão robustos e escaláveis quanto os reativos. Por isso são utilizados para formulários simples, que não necessitam de uma lógica complexa por trás e que possam ser gerenciados isoladamente no template.

No Angular existem algumas maneiras de se criar formulários reativos. Uma delas é atribuir ao formulário uma instância da class FromGroup, e para cada input uma instância da classe FormControl:

import { FormGroup, FormControl, Validators } from '@angular/forms';
export class Classe implements OnInit {

  form!: FormGroup;

  ngOnInit(): void {
    this.form = new FormGroup({
      content: new FormControl('',[Validators.required]),
      author: new FormControl('',[Validators.required]),
      model: new FormControl('')
    })
  }
}

Ou utilizar uma instancia de FormBuilder, um objeto construtor de formulários, que por baixo dos panos vai criar e organizar as instâncias de FormGroup e FormControl, porem utilizando uma sintaxe simplificada:

import { FormGroup, FormBuilder, Validators } from '@angular/forms';

export class Classe implements OnInit {

  form!: FormGroup;

  constructor(private formBuilder: FormBuilder) { }

  ngOnInit(): void {
    this.form = this.formBuilder.group({
      content: ['',[Validators.required]],
      author: ['', [Validators.required]],
      model: ['', [Validators.minLenght(10)]]
    })
  }
}

Também é possível utilizar a validadores, através da classe Validators, do pacote angular.forms, para fazer a validação dos dados de entrada do formulário, adicionando na definição do FormControl um array de Validators com as condições necessárias

Para utilizá-los é necesário importar o pacote ReacitveFormsModule, do pacote angular.forms no arquivo app.module.ts

#estudar

Para utilizar mais de um Validator em um FormControl é necessário usar o método Validators.compose(), passando como parâmetro um array de objetos Validators:

Validators.composeAsync() -> Compõe vários validadores assíncronos em uma única função que retorna a união dos objetos de erro individuais para o controle fornecido.

  this.form = this.formBuilder.group({
    content: ['',
      Validators.compose([
        Validators.required,
        Validators.minLenght(),
        Validators.pattern()])]
  })

Alguns validators importantes:

  • Validators.min() e .max() -> Validador que determina o valor mínimo ou máximo do número no controle
  • Validators.max() -> Validador que exige que o valor do controle seja menor ou igual ao número fornecido.
  • Validators.requiredTrue() -> Validador que exige que o valor do controle seja verdadeiro. Este validador é comumente usado para caixas de seleção obrigatórias.
  • Validators.required -> Validador que torna o campo obrigatório.
  • Validators.email() -> Validador que exige que o valor do controle passe em um teste de validação de email.
  • Validators.maxLength() e .minLenght() -> Validadores que determinam o tamanho mínimo e máximo de um controle de texto.
  • Validators.nullValidator() -> Validador de valores nulos.
  • Validators.pattern() -> Validador que verifica um controle com base numa expressão regular.

Validators personalizados
...

É possível criar regras de validação personalizadas e utilizá-las juntamente às regras de validação built-in do Angular. Para isso é necessário criar uma função (de preferência num arquivo separado do componente), que receba um AbstractControl para representar o FormControl. Também é possível criar esse validador como uma Diretiva Personalizada.
Essa função deve conter a lógica de negócio da validação e retornar um booleano ou null, como no exemplo abaixo, que verifica se o texto contém apenas letras minúsculas:

import { AbstractControl } from "@angular/forms";

export function minusculoValidator(control: AbstractControl) {
    const texto = control.value as string;
    if(texto !== texto?.toLowerCase()) {
        return { minusculo: true };
    } else
    return null;
}

Essa função deve ser importada no componente, e adicionada junto aos demais validadores no array do Validators.compose:

import { minusculoValidator } from './minusculoValidators';
(...)
  this.formBuilder.group({
    texto: ['', 
      Validators.compose([
        Validators.required,
        Validators.minLength(3),
        minusculoValidator])
    ]
  })
O retorno da função será o objeto usado para fazer a verificação de formulário válido.
<div class="mensagem__erro"
     *ngIf="formulario.get('texto')?.errors?.['minusculo'] && formulario.get('texto')?.touched">
       Esse campo aceita apenas letras minúsculas
</div>

Validadores assíncronos
...

Quando se está trabalhando com requisições é necessário esperar a resposta, de forma assícrona. Para fazer a validação das respostas nesse caso existe o grupo de diretivas NG_ASYNC_VALIDATORS.

Por exemplo, na consulta a um CEP pela API do ViaCEP, poderia haver o seguinte validador: 

import { Directive } from '@angular/core';
import { Observable } from 'rxjs';
import{ AsyncValidator, NG_ASYNC_VALIDATORS} from '@angular/forms';

@directive({
  selector: '[appValidandoCep]',    // -> o seletor será usado para aplicar o validator
  providers: [{
    provide: `NG_ASYNC_VALIDATORS`,
    useExisting: ValidandoCepDirective,
    multi: true
  }]
})
export class ValidandoCepDirective implements AsyncValidator{

  constructor(private service: ConsultaCepService){
    validate(control: AbstractControl): Promise<ValidatorsErrors | null> | Observable<ValidatorsErrors | null> {
      const cep = control.value;
      
      return this.service.getConsultaCep(cep).pipe(map(
        (resultado:any) => resultado.erro ? {'validadorCep': true} : null
      ))
    } 
  }
}

O validator pode ser chamado diretamente no elemento input do template ou no array de validadores do FormControl no componente.

<input name="cep" type="text" appValidandoCep>
Código do serviço
import { HttpClient } from '@angular/common/http';
import { Injectable } from '@angular/core';

@Injectable({
  provideIn: 'root'
})
export class ConsultaCepService{
  url_API = 'https://viacep.com.br/ws/'

  constructor(private http: HttpClient) {
  }
  getConsultaCep(cep: string){
    return this.http.get(`${url_API}${cep}/json`,)
  }
}

Lidando com erros
...

O formulário construído retorna os erros que ocorrerem na validação, no formato de JSON, que pode ser acessado para identificar cada tipo de erro:

  json_de_erros = form.get('content')?.errors
  erro_de_required = form.get('content')?.errors?.['required']

Formulário "touched"
...

Para exibir erros, avisos, etc. para o usuário, após ele interagir com um determinado campo, existe a propriedade touched, que pode ser acessada no form:

  isTouched = form.get('content')?.touched
