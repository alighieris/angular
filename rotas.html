Rotas

Routers funcionam como espaços reservados que podem ser alterados dinamicamente pelo Angular.
Aqui é aplicado o conceito de single-page app, alterando, ocultando ou mostrando componentes particulares, ao invés de fazer uma nova comunicação com o servidor para obter uma página completa. Os users navegam entre diferentes views definidas.

Para lidar com a navegação entre as views, o Angular Router interpreta as mudanças na URL do browser como uma instrução para alterar a view.

Para utilizar uma rota no Angular, utilizam-se tags especiais, como <router-outlet>.
Essa tag reserva o espaço no HTML para os componentes dinâmicos, que serão configurados a partir do arquivo de rotas src/app/app-routing.module.ts.
Neste arquivo vão ser configurados os endpoints/paths das diferentes páginas, e o HTML exibirá o componente equivalente àquela rota.

No arquivo app-routing.module.ts:

    // imports (...)
    const routes: Routes = [
        {
            path: 'path_desejado_na_url',
            component: ClasseComponente1
        }
        // É possível também fazer o redirecionamento a partir de padrões de página, para uma mesma rota já estabelecida
        {
            path: '',
            redirectTo: 'path_desejado_na_url',
            pathMatch: full     // avalia todo o conteúdo da url
            // ou
            pathMatch: prefix // avalia apenas o conteúdo ddo domínio até o path especificado, ignora o que vem depois
        }
    ];

Com o path especificado nas Routes, é possível utilizar a rota em elementos, a partir da diretiva routerLink:

    <button routerLink="path_desejado_na_url">Acessar Rota</button>

RouterModule
...

O RouterModule é um módulo Angular que permite definir rotas dentro da aplicação. Disponibilizando diferentes estratégias de correspondência de caminho, fácil acesso aos parâmetros de rota e proteções de rota para componentes que devem ter acesso restrito.
Este módulo dispõe de diretivas que permitem o uso das rotas e alteram o seu funcionamento e dos elementos/componentes que as utilizam.

O objeto Route também pode ser injetado como dependência num componente, para ser usado em métodos que precisarão redirecionar/navegar entre diferentes telas.

(...)
import { Router } from '@angular/router'
(...)
export ClassComponent implements OnInit {
	constructor(private router: Router) { }

	redirect(){
		this.router.navigate(['/path'])
	}
}
