Services, Dependências e Backend

Dependências são serviços, objetos, funções ou até mesmo valores que uma classe necessita para desempenhar sua função. Por exemplo, uma classe A pode precisar de informações/métodos de uma classe B para poder funcionar. Logo B é uma dependência para A.
Nesse caso a classe A pode ser a responsável por criar uma instância da classe B, ou então essa dependência pode ser passada para ela, ou seja, injetada nela.

A Injeção de Dependências (DI) é um padrão de projeto no qual a classe solicita dependências de fontes externas ao invés de criá-las.

Services
...

É uma boa prática que os componentes contenham apenas a lógica para definir comportamentos e conseguir renderizar os arquivos na tela, dessa forma, regras de negócio e métodos de comunicação com o servidor devem ficar isolados da definição dos componentes. Estes são os services.

Para criar um serviço injetável no Angular, utiliza-se o comando:

ng generate service <path/nome_do_serviço>
# Ou a forma abreviada:
ng g s <path/nome_do_serviço>

A classe criada possui o decorator @Injectable(), que indica para o Angular que essa classe é injetável e pode ser utilizada por outras classes. Por padrão esse decorator possui o metadado provideIn que determina para que parte da aplicação o serviço será injetado. No exemplo, o serviço será provido para toda a aplicação, pois o metadado foi setado para a raiz do projeto:

import { Injectable } from '@angular/core';

@Injectable({
	provideIn: 'root'
})

export class MyService {
constructor() {}

metodo1() { (...) }
}

Injetando serviços em componentes
...

Sem a injeção de dependências, seria necessário instanciar todas os serviços necessários para o funcionamento de um componente manualmente, além de passar todos os parâmetros necessários para isso, por exemplo:


 export class myComponent implements OnInit {
	 (...)
	 
	 service1 = new Service1();
	 service2 = new Service2(this.param1, this.param2);	 
 }

Isso geraria acoplamento de classes e repetição de código desnecessários. Utilizando a injeção de dependências, esse processo se torna bem mais simples:

(...)
import { MyService } from 'src/app/services'
 
export class MyComponent {
	constructor(private myService: MyService){ }
}

A Injeção de Dependências no Angular é feita via construtor, onde especificamos uma propriedade com o tipo da dependência.

Acesso ao backend
...

Para acesso ao backend, fazendo requisições Http a um servidor, é necessário usar o HttpClient, disponível no pacote angular.common.http.
Para isso, é necessário injetar/instanciar um objeto do tipo HttpClient na classe do serviço e utilizar seus métodos para fazer as requisições.

(...)
import { HttpClient } from '@angular/common/http'
import { Observable } from 'rxjs'

@Injectable({
	provideIn: 'root'
})
export class MyService{
	private readonly urlAPI = 'http://myapi.com/endpoint'

	constructor(private http:HttpClient) {}

	getRequest(): Observable {
		return this.http.get(this.urlAPI)
	}
}
Para utilizar o HttpClient é necessário importar o módulo HttpClientModule no arquivo app.module.ts.

Porém, a resposta dos métodos do HttpClient, são na verdade Observables, e não um JSON

Observables
...

Observables são objetos similares às promises do JavaScript, porém com a diferença de que eles podem ter transferência de dados contínuas, emitindo dados várias vezes durante a sua existência.

Observables fazem parte da biblioteca RxJS, que é utilizada pelo Angular "por baixo dos panos", e já vem instalada no Angular. Então para trabalhar com eles, é necessário importá-los no componente/serviço onde será usado:

import { Observable } from 'rxjs'

Na prática, Observables são interfaces que lidam com uma variedade de operações assíncronas comuns, como requests do módulo HTTP, e eventos de interação de usuário pelos módulos Router e Forms.

O uso consiste em basicamente se inscrever à um Observable, informando o que deverá ser feito com o dado que irá receber. A inscrição pode escutar 3 tipos de interações:

  • Next -> notificação de sucesso, em que um valor é passado corretamente pela stream
  • Error -> notificação de falha, que levanta uma exception ou JavaScript Error. Encerra a execução.
  • Complete -> notificação de que a stream de dados foi completada, e será encerrada. Não envia dados.

Resumidamente, Observables permitem:

  • Receber dados várias vezes e em vários momentos
  • Nos inscrever para receber dados de um mesmo Observable em vários pontos da aplicação
  • Executar alguma operação quando o dado for recebido com sucesso
  • Executar alguma operação quando emitir erro
  • Executar alguma operação quando completar

Ouvindo um Observable
...

Para receber a stream de dados de um Observable, é necessário fazer a inscrição no objeto, e definir o que acontece com os dados recebidos. Assumindo o serviço definido acima, um componente que receba este serviço com dependência poderá usar:

export ClassComponent implements OnInit {
	public data: DataInterface[] = []
	constructor(private service: MyService) { }

	ngOnInit(): void {
		this.service.getRequest().subscribe((dataReceived) => {
			this.data = dataReceived
		})
	}
	// Obs: pode ser usado no ngOnInit ou qualquer outro método.
}
