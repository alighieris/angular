Diretivas e Elementos de Sintaxe

Diretivas são classes que tem a capacidade de alterar os elementos no template, adicionando ou modificando o comportamento dos elementos. Na prática, são funções que serão executadas quando encontradas no DOM pelo compilador Angular.
Diretivas parecem atributos HTML e são colocados no template, de forma a acessar o elemento/componente para operar algum tipo de lógica, dando flexibilidade ao código e permitindo que o template se torne dinâmico.

Declaração de uma diretiva:

import { Directive } from '@angular/core';

@Directive({
  selector: '[minhaDiretiva]'
})
export class MinhaDiretiva {
  constructor() { }
}

Diretivas Estruturais
...

Este tipo de diretiva executa alterações no layout do DOM, adicionado ou removendo elementos do DOM.

As principais Diretivas Estruturais, built-in no Angular são:

*ngIf
...

Utiliza uma condição boolena para exibir ou esconder elementos.

<p *ngIf="variable">Este parágrafo será exibido se a variável 'variable' for verdadeira.</p>

Também é possível criar condições de if else, e trazer outros elementos/componentes para o lugar caso a condição não seja atendida:

<div *ngIf="variable, else templateCaseFalse">
	Este template será inserido caso a variável seja verdadeira
</div>

<ng-template #templateCaseFalse>
  <div>
  Este template será inserido caso a variável seja falsa
  </div>
</ng-template>
O elemento ng-template não será renderizado no HTML fonte. Ver ng-template

*ngFor
...

Exibe elementos com base na iteração em uma lista ou coleção de dados.

<p *ngFor="let item of lista">Cria um elemento "p" para cada {{item}} da lista</p>

*ngSwitch
...

Permite alterar entre diferentes elementos pré-definidos com base em uma expressão lógica.

<div [ngSwitch]="valor">
    <p *ngSwitchCase="'a'">O valor é a</p>
    <p *ngSwitchCase="'b'">O valor é b</p>
    <p *ngSwitchDefault>O valor não é nem a nem b</p>
</div>

Diretivas de Atributos
...

Diretivas que fazem mudanças no comportamento ou aparência de elementos, por exemplo alterando seu estilo ou classes CSS.

As principais Diretivas de Atributo built-in no Angular são:

ngClass
...

Usado para adicionar, remover, ou condicionar o uso de classes CSS a um elemento, a partir de expressões lógicas.

<!-- é possível atribuir diretamente classes, ou a partir de variáveis  -->
<div [ngClass]=" classe1 {{variable.class}}">Hello World!</div>

<!-- para condicionais, o elemento receberá a classe "classe-ativa" caso o valor da "variable" ou da expressão seja true -->
<div [ngClass]="{'classe1': variable, 'classe2': param1 > param2}">Olá, mundo!</div>

<!-- também é possivel utilizar operador ternário -->
<div [ngClass]="variable ? 'classe-ativa' : 'classe-inativa'">Hello World!</div>

<!-- ou chamada de funções que retornem um booleano, ou o nome da classe -->
<div [ngClass]="'classe-ativa': 'isActive(var)'">Hello World!</div>
<div [ngClass]="retornaClasse(var)'">Hello World!</div>

ngStyle
...

Funciona de maneira similar à ngClass, mas trabalhando diretamente com as propriedades inline ao invés de classes completas:

<div [ngStyle]="{'background-color': corDeFundo}">Olá, mundo!</div>

Também é possível passar um JSON completo como parâmetro, contendo todas as propriedades e valores necessários. Inclusive aproveitando propriedades e métodos dentro da classe do componente para trabalhar as condicionais:

// (...) Definição do objeto no Typescript
this.currentStyles = {
    'font-style': this.canSave ? 'italic' : 'normal',
    'font-weight': !this.isUnchanged ? 'bold' : 'normal',
    'font-size': this.isSpecial ? '24px' : '12px',
}

No HTML basta passar a variável:

<div [ngStyle]="currentStyles">
  This div is initially italic, normal weight, and extra large (24px).
</div>

ngModel
...

Essa diretiva cria uma instância de um FormControl, que serve para fazer o binding entre propriedades de um componente e elementos de um formulário, criando assim um Two way biding.

Essa diretiva faz parte do pacote angular.core, e precisa ser importada no arquivo app.module.ts, juntamente com o FormsModule, do angular.forms.

Diretivas de Componentes
...

Tecnicamente, um Componente Angular é uma diretiva que contém um template. Essa é a única diferença entre este tipo de diretiva e as demais.
Ela é declarada com o decorator @Component, e contém um trecho de HTML (template), que é o que define a UI do componente.

@Component({
  selector: 'meu-componente',
  template: '<p>Olá, { {nome}}</p>'
})
export class MeuComponente {
  nome = 'Angular';
}

Diretivas personalizadas
...

#estudar

É possível criar diretivas personalizadas, que podem ser aplicadas de diversas formas ao código, como por exemplo funcionando como validadores de formulário, classe CSS ou propriedade de elementos HMTL

Elementos de Sintaxe Angular
...

Estas não são diretivas, nem tags HTML padrão, mas sim elementos de sintaxe do Angular. Esses elementos não serão renderizados no DOM (ou seja, na prática não serão criados novos elementos/nós), mas podem ser usados para trabalhar a lógica e condicionais de exibição de outros elementos.

ng-template
...

Cria um modelo de bloco que pode ser reutilizado em outras partes do código, quando necessário. O bloco pode ser definido em qualquer parte do arquivo html fonte, porém não será renderizado para o usuário, apenas o conteúdo, caso as condições sejam atendidas.
Esse template pode ser chamado em outros elementos através do "ID" definido com #nome_do_template, usando a diretiva *ngTemplateOutlet*:

  <ng-template #myTemplate>
   <div *ngIf="hasBalance">
     <p>Seu saldo é R$ 100,00.</p>
   </div>
   <div *ngIf="!hasBalance">
     <p>Desculpe, você não tem saldo.</p>
   </div>
 </ng-template>
 
 (...)
 <!-- fazendo a chamada do template em outro trecho do código-->
 <ng-container *ngTemplateOutlet="myTemplate"></ng-container>

ng-container
...

Atua como um "wrapper" para englobar trechos de html que precisam passar por uma condicional ou atuação de uma diretiva estrutural. Como o elemento não é renderizado, ele serve apenas para aplicar a condição necessária.
No exemplo, não será criado um elemento "ng-container" ao renderizar a tela, apenas os elementos "p", caso o valor de variable seja true:

 <ng-container *ngIf="variable">
   <p>Este é o primeiro parágrafo</p>
   <p>Este é o segundo parágrafo</p>
 </ng-container>

ng-content
...

Permite que um componente pai injete conteúdo num template de um componente filho. No componente filho, define-se um "placeholder" para o conteúdo que será inserido na chamada do elemento HTML filho. Dessa forma é possível posicionar e trabalhar o elemento injetado dentro do template filho dinamicamente.

Exemplo da aplicação do ng-content, iniciando pela definição do template do componente filho:

import { Component, OnInit } from '@angular/core';

@Component({
selector: 'multiples',
styleUrls: [`./multiples.component.scss`],
template: `
	<div>
		<h1>
			<ng-content select="[title]"></ng-content>
		</h1>
		<p>
			<ng-content select="[description]"></ng-content>
		</p>
	</div>
`})
export class MultiplesComponent implements OnInit {
	constructor() { }
		ngOnInit() { (...) }
}

As tags ng-content posicionarão os elementos chamados dentro do componente no html principal. Sendo possível também selecionar por nome cada tag. No template do elemento pai, seria feita a seguinte inserção:

(...)
<multiples>
	<span title>Título</span>
	<span description>Descrição</span>
</multiples>

Dessa forma o componente filho "multiple" poderá ter conteúdo injetado nas posições definidas e usando as características e estilos definidos internamente.
