(RxJS) Operators

Operadores
...

Operadores são funções que permitem combinar e manipular Observables para obter diferentes tipos de respostas.
No RxJS existem dois tipos de Operadores:

  • Pipeable Operators: tipo de operadores que podem ser organizados em um "pipe", ou seja uma sequência de operações pelos quais os dados passam.
  • Creation Operators: operadores "standalone" que disponibilizam métodos para criar novos Observables.
RxJS - Operators

Pipeable Operators
...

A Pipeable Operator is a function that takes an Observable as its input and returns another Observable. It is a pure operation: the previous Observable stays unmodified.

Ou seja, quando um Pipeable Operator é chamado, ele não altera a instância do Observable existente, mas retorna um novo Observable que contém uma lógica de inscrição baseada no primeiro Observable.

A Pipeable Operator is essentially a pure function which takes one Observable as input and generates another Observable as output. Subscribing to the output Observable will also subscribe to the input Observable.

Ou seja, a inscrição ocorre nos dois Observables, a instância original e no output do operador.

A Pipeable Operator Factory is a function that can take parameters to set the context and return a Pipeable Operator. The factory’s arguments belong to the operator’s lexical scope.

Pipe
...

O método pipe permite fazer a convolução de diversos operadores de maneira simplificada para os observables. Ao invés de ter um encadeamento de chamada de operadores e respostas, é possível chamá-los isoladamente e definir suas lógicas de maneira mais organizada: 

myObservable.pipe(
	op1(),
	op2(),
	op3(),
	op4()
)

// Ao invés de um encadeamento de chamadas como:
op4()(op3()(op2()(op1()(myObservable))))

Dessa forma, é possível criar uma sequência de operações a serem executadas na stream de dados, e ao final será gerado um output com um novo observable, no qual se pode fazer uma inscrição. O observable original não será alterado no processo e pode ser usado em outras partes do código. 

observable2 = myObservable.pipe(
	op1(),
	op2()
)

myObservable.subscribe(result1 => console.log('Resultado do observable original:', result1))
observable2.subscribe(result2 => console.log('Resultado do novo observable:', result2))

Tap
...

O operador tap permite implementar "efeitos colaterais" na emissão de dados do observable sem que se modifique os valores ou a execução em si. É comumente usado para debug, log ou disparar ações que não afetam a stream de dados. 

myObservable().pipe(
	tap(value => console.log('Value:', value)),
	map(value => value * 2)
).subscribe(result => console.log(result));

Supondo que os valores passados na stream sejam 1, 2, 3, ..., o resultado esperado é: 

Value: 1
2
Value: 2
4
Value: 3
6
(...)

Map
...

O operador map (ver mapping) é uma função de alta ordem permite transformar os dados emitidos por um observable, aplicando uma função/regra sobre uma coleção e retornando um novo objeto. 

myObservable().pipe(
	map(value => value * 2)
).subscribe(result => console.log(result));

Filter
...

O operador filter permite filtrar seletivamente os valores emitidos por um observable, baseado numa condição dada. Ele recebe uma função determinada como argumento e apenas emite valores que satisfaçam a condição da função: 

myObservable().pipe(
	filter(value => value % 2 === 0) // retorna apenas valores múltiplos de 2
).subscribe(result => console.log(result));

Debounce Time
...

O operador debounceTime é usado para controle da frequência de emissão de valores pelo observable. Ele faz com que o pipe "espere" uma certo intervalo de tempo antes de emitir o valor da stream atual: 

myObservable().pipe(
	debounceTime(300)    // passa o valor da stream após 300 ms
).subscribe(result => console.log(result));

Switch Map
...

O operador switchMap geralmente é usado para lidar com operações assíncronas (requests HTTP, por exemplo). Esse método transforma os valores emitidos por um observable em um novo observable, "trocando" os valores emitidos, e automaticamente encerra a inscrição no observable novo, quando um novo valor chega para que possa ser criado um outro. 

const button = document.getElementById('button');  

// O observable original que gera o pipe é criado a partir de um evento de click em um botão
fromEvent(button, 'click').pipe(
// O switchMap executa uma outra função, deixando de passar o "valor do click" pela stream, e passando o novo observable obtido pela requisição HTTP
  switchMap(() => fetchData())  
).subscribe(data => {  
  // O "data" recebido aqui, é resultado da inscrição no observable gerado pela requisição HTTP, e não pelo observable do click no botão 
});  
  
function fetchData() {  
  // Função que executa o request HTTP e retorna um Observable  
}
