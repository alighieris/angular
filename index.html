0. Angular

Angular surgiu como um framework baseado em JS criado em 2009, dentro da Google. A primeira versão, Angular, ficou bastante popular, e mais tarde em 2016 foi reescrita para uma nova versão que alterou boa parte do funcionamento original do framework, introduzindo o Typescript como linguagem praticamente obrigatória, programação reativa e vários outros conceitos. A nova versão desagradou boa parte da comunidade, pois mudava completamente a forma com que o Angular funcionava.
Dessa forma, os frameworks foram separados, a primeira versão passou a ser chamada de AngularJS, e a segunda apenas de Angular. Estabelecendo assim dois suportes e linhas de desenvolvimento diferentes para cada um dos frameworks

Hoje, Angular não é apenas um framework, mas um ecossistema completo de desenvolvimento de projetos, contendo bibliotecas próprias e toda uma plataforma que contém as necessidades comuns de desenvolvimento de interfaces e ferramentas para a a construção de um projeto.

Como framework, Angular é baseado em componentes, que são trechos de código reutilizáveis escritos em HTML, 0. CSS e TypeScript. Geralmente representam um elemento da interface.
Em Angular, o HTML possui propriedades especiais, como diretivas e interpolação, que permitem construir conteúdo dinâmico usando comportamentos e valores definidos na classe do componente escrita em . Dessa forma, a biblioteca é popular por ser muito conectada a conceitos como separação de responsabilidades, padrões de projetos e arquitetura modular.
Os componentes desenvolvidos podem ser testados, usando frameworks como Jest e Jasmine, sendo o último o padrão em projetos Angular gerados pela Angular CLI.

Componentes
...

Um Componente Angular é uma combinação de uma view (um trecho de HTML) e uma lógica (uma classe Typescript) que controla os dados e o comportamento dessa View.
É responsável por capturar a entrada do usuário, processando as interações de acordo com as regras de négocios e atualizar a view de acordo com esse processamento.

Tecnicamente, em Angular, um Componente é uma diretiva que contém um Template (HMTL).

Diretivas e Elementos de Sintaxe
...

Diretivas são classes que tem a capacidade de alterar os elementos no template, adicionando ou modificando o comportamento dos elementos. Na prática, são funções que serão executadas quando encontradas no DOM pelo compilador Angular.
Diretivas parecem atributos HTML e são colocados no template, de forma a acessar o elemento/componente para operar algum tipo de lógica, dando flexibilidade ao código e permitindo que o template se torne dinâmico.
Podem ser criadas, mas os pacotes Angular já possuem várias Diretivas para que ações comuns possam ser feitas com menos código.

Angular CLI
...

É uma ferramenta que permite trabalhar em aplicações Angular com mais praticidade. Criar projetos, criar artefatos e outras ações rotineiras são facilitadas pela CLI.

O CLI é uma aplicação que roda no terminal e oferece comandos para uso no projeto, porém não é parte do framework, apenas outra solução que faz parte do ecossistema Angular.
