One way data biding
...

Forma de passar o valor de variáveis e parâmetros dos componentes para o HTML exibir.
Quando passado como propriedade do elemento HTML, a propriedade deve ser chamada entre colchetes ([]) e o nome do parâmetro entre aspas:
Quando passado diretamente para o conteúdo do elemento HTML, deve-se usar a sintaxe de chaves duplas ({{ }}).

Exemplo:

Na definição da classe do componente em TS:

import { Component, OnInit } from '@angular/core';

@Component({
  selector: 'app-create-object',
  templateUrl: './create-object.component.html',
  styleUrls: ['./create-object.component.css']
})
export class CreateObjectComponent implements OnInit {

  object = {
    id:'1',
    content: 'Aprendendo Angular',
  }

  constructor() {
    // (...) 
    }

  ngOnInit(): void {
  }

}

No HTML:

    <input
        type="textarea"
        placeholder = "Lorem ipsum..."
        [value] = "object.id">
    
    <p>{{ object.content }}</p>
Interpolação

É possível usar as chaves duplas para fazer interpolação de variáveis em diversas situações, como passar valores para as propriedades HTML ou conteúdo de texto dentro do elemento, bem como outros tipos de parâmetros, como definir as classes CSS de um elemento.

Exemplo:

No TS do componente:

// Implementações do componente (...)
export class ClasseComponente implements OnInit {

  propriedade = {
    conteudo: 'Hello World!',
    classeCSS: 'modelo1',
    tipo: 'submit'
  }

No HTML:

    <button type="{{ propriedade.tipo }}" class="{{ propriedade.classeCSS }} modelo2">
        O conteúdo vai aqui: {{ propriedade.conteudo}}
    </button>

Two way data biding
...

Forma de interligar as propriedades dos objetos TS com as informações registradas no componentes Angular através de interação do usuário. Ao contrário do one way (que permite passar propriedades dos componentes para os templates), aqui é possível fazer os dois caminhos.

Para fazer esse tipo de biding, é usada a Diretiva Angular ngModel.
Dessa forma, os dados presentes no template, resultantes da interação do usuário, estarão disponíveis dinamicamente para processamento pelo componente, ou seja, no código Typescript.

Como ao criar um formulário, com um campo de input no template, é possível criar um ngModel e referenciar uma propriedade/objeto da classe do componente para fazer o bind das informações:

<input
  type="textarea"
  [(ngModel)] = "objeto.propriedade"
/>
<button (ngClick)="printar()">Processar</button>

E fazer o seu processamento no componente, usando código Typescript:

(...)
export class ClasseComponente implements OnInit {
	objeto = {
		propriedade: 'conteudo_inicial'
	}
	
	printar(): void {
		console.log(this.objeto.propriedade)
	}
}

O typescript poderá ter um valor pré-definido que será passado incialmente para o Template, e conforme o usuário interaja com o template e altera o valor de alguma forma, esse valor é repassado de volta para o TS, e pode ser processado, por exemplo através do método printar().

Permite escutar e responder ações e interações do usuário com os componentes e elementos da aplicação.
O biding de evento é adicionado diretamente no elemento HTML, como uma propriedade indicada entre parênteses como no exemplo:

<button (click)="botaoClick()">Botão</button>

@Input
...

#estudar
É possível definir uma propriedade do componente que será "bindável", a partir do TS, de forma que o componente Pai possa acessar a propriedade de um elemento Filho, através da chamada da propriedade do HMTL. Para isso usa-se o decorator @Input na definição dessa propriedade.

Angular - Inputs
import { Input } from '@angular/core' 
@Component({
  selector: 'componente-filho',
  templateUrl: './componente-filho.component.html',
  styleUrls: ['./componente-filho.component.css']
})
export class ComponenteFilho implements OnInit {

  @Input()
  property = {
    content: 'Hello World!',
    type: 'tipo1'
  }
}

O Binding passa a ser possível no template do componente Pai, na chamada no Filho:

<componente-filho [property] = "propriedadeDoPai"> </componente-filho>

Isso permite a passagem de valores das propriedades de um componente pai para o filho, uma vez que o valor de propriedadeDoPai pode ser recebido de uma propriedade pai do elemento componente-filho. Dessa forma é possível, fazer a passagem de valores dinamicamente entre os componentes da aplicação.

@Output
...

#estudar
Similar ao decorator @Input, o @Output permite o bind de propriedades dos componentes, mas permitindo a passagem de informações do componente Filho para o componente Pai.

No componente filho, é necessário criar uma propriedade Output, com o tipo EventEmitter que será o emissor do evento para o pai. Também será necessário criar um método onde o emit será chamado, e este método será o parâmetro utilizado no template para fazer a chamada.

import { Output, EventEmitter } from '@angular/core' 
(...)
@Component({
  selector: 'componente-filho',
  templateUrl: './componente-filho.component.html',
  styleUrls: ['./componente-filho.component.css']
})
export class ComponenteFilho implements OnInit {
  propriedadeEmitida: Propriedade ={
    (...)
  }

  @Output()
  emissorEvento = new EventEmitter()

  metodoEmitirEvento(){
    this.emissorEvento.emit(propriedadeEmitida)
  }
}

No componente pai, também é necessário qe haja um método preparado para receber o objeto emitido pelo filho:

export class ComponentePai implements OnInit {
  (...)
  metodoReceberEvento(propriedadeEmitida: Propriedade){
    console.log(`Faz qualquer coisa com a ${propriedadeEmitida}`)
  }
}

Por fim, no template do componente pai, ao chamar o componente filho, é possível chamar o objeto de Output do filho como um parâmetro de event biding, e atribuir o método de recebimento do pai à execução desse evento, recebendo como parâmetro o $event emitido pelo filho.

  <componente-filho (emissorEvento)="metodoReceberEvento($event)"></componente-filho>
